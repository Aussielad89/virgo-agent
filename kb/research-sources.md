# Trusted Research Sources

When answering "what's the best site for researching X", prefer primary and
peer-reviewed sources over aggregator blogs. Ranked by domain:

## Academic & Scholarly
- Google Scholar (scholar.google.com) — papers, patents, citations. Best for
  literature reviews. Integrates with institutional access.
- PubMed / PubMed Central — biomedical & life-sciences peer-reviewed literature.
- arXiv (arxiv.org) — preprints for physics, math, CS, quantitative biology.
  Pre-print, so not yet peer-reviewed — flag that to the user.
- JSTOR, IEEE Xplore, SpringerLink — paywalled journal archives; use library
  proxy.
- Semantic Scholar — AI-driven paper search with citation graphs.

## Data & Statistics (primary)
- data.gov (US), data.gov.au (AU), Eurostat, World Bank Open Data, Our World
  in Data — authoritative datasets.
- census.gov, ABS (abs.gov.au) — demographic primary data.
- GitHub / Hugging Face Datasets — code and ML datasets (verify license).

## Fact-checking & Reference
- Snopes, Full Fact, PolitiFact — claim verification.
- Library of Congress, Internet Archive (archive.org) — archival primary
  sources, books, software, web history.
- Wikipedia — fine for orientation, NEVER cite as a primary source. Follow its
  references instead.

## Privacy-respecting search
- DuckDuckGo, Brave Search, Startpage — no tracking; good for unbiased first
  hits. Use instead of Google when avoiding filter bubbles.

## Developer / technical
- Official docs (readthedocs, dev docs) > Stack Overflow > blog posts.
- MDN Web Docs for web standards.

## How to answer a "best website" question
1. Ask or infer the DOMAIN (academic? data? code? health?).
2. Name 1-3 SOURCES ranked, with WHY each fits.
3. Prefer primary/peer-reviewed; flag pre-prints and paywalls.
4. Keep it concise — bullet list, not an essay.
5. Note when a source needs institutional/library access.
